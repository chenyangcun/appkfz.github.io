使用Nodejs实现聊天服务器

发表于 2015-06-13   |   分类于 web   |  

两年前在项目中使用nodejs+socket.io+redis实现的聊天和推送服务器,
基本上几百行代码就实现了整个功能，在项目中单服务器单进程可以跑到支持
5000人左右同时在线。

主要思路

  1. 用户上线后，根据用户的userid和socket，保存到一个全局的map中
  2. 发送消息时，根据对方的userid找到对应的socket，通过socket.emit发送消息给对方
  3. 如果对方不在线，在redis中保留消息，在对方上线后推送给用户
  4. 用户下线后，从全局的map中删除对应的用户socket
  5. 由于需要保持长连接，客户端需要定时发心跳给服务端，所以定义了一个心跳消息，可以5分钟发一次

eclipse项目迁移到Android Studio

发表于 2015-06-12   |   分类于 Android   |  

由于公司项目需要多包名APK的支持，所以把公司的项目从eclipse迁移到了Android Studio.

以下是一些迁移经验：

迁移准备

  • Android Studio对代码检查比较严格，代码中如果存在一些重复ID和重复String，尽量都使用lint处理掉。
  • Android Studio对于JAR包重复会导致编译失败，如果引用的项目中有相同的jar包，先清理掉，只保留一份。

开始迁移

目前Android Studio支持两种方式迁移

  • 如果想保持原有的目录结构，
    在eclipse继续开发，我们采用通过eclipse导出gradle文件的方式，
    在Studio中再import gradle文件的方式加进来
  • 如果不准备保留原有结构，直接用Android Studio打开eclipse项目，会提示迁移，该方式是通过拷贝的方式，新生成一份新结构的代码。

完成迁移

在导入gradle文件后，每个Module都有一个build.gradle文件，
在setting.gradle中会include所有需要Module。
导入完成后，尝试编译运行一下，首次运行编译较慢，需要耐心等待。
